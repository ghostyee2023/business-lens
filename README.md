# business-lens · 商机透镜

> *「看清一个机会的商业逻辑」*

<br>

**帮你判断一个产品机会到底有没有商机：谁会买、为什么买、预算从哪里来、先验证什么。**

当前版本：**v0.2.0**。现已支持 `idea-opportunity/v1 → business-opportunity/v1 → maker-forge` 的可审计交接。

不是把每个好想法都包装成“商业机会”。<br>
是把用户、购买者、预算、获客、交付和致命假设一层层照出来。

## 装上就能用

打开你正在用的 AI agent，告诉它：

```text
帮我安装这个 skill：https://github.com/ghostyee2023/business-lens
```

或者用通用 CLI 安装器：

```bash
npx skills add ghostyee2023/business-lens
```

装好后直接说话：

```text
用 business-lens 判断这个想法有没有商机：给知识博主做一个从长文生成图卡选题的工具。
这个产品谁会买？预算从哪里来？商业逻辑通不通？

```

如果自动加载失败，手动把 `SKILL.md` 的内容粘贴进对话也能用。它本质就是一份 markdown + YAML frontmatter。

## 能做什么

商机透镜负责“商业逻辑判断”，它位于灵感和 MVP 之间。

| 能力 | 说明 | 产出 |
| --- | --- | --- |
| **商业逻辑链** | 从用户、痛点、替代方案、购买者、获客和交付完整检查 | 一条可审视的商业逻辑链 |
| **购买者判断** | 区分谁使用、谁付费、谁拥有预算 | buyer / budget owner |
| **商机评分** | 评估痛点强度、预算匹配、获客可达、交付可重复 | 结构化分数 |
| **商业模式选择** | 判断适合模板、服务辅助 MVP、订阅、插件、数据产品等 | 第一版商业模式 |
| **致命假设** | 明确什么事实会让这个想法不成立 | 最大风险 |
| **验证动作** | 强制写清真实请求、信号、通过/失败阈值和截止时间 | 可执行实验 |
| **证据分层** | 区分观察、内部/外部证据、类比、推断、假设和反证 | 可审计依据 |
| **结构化决策** | 用 kill gates 拦截虚假 proceed，并校验重复 ID | `business-opportunity/v1` |

## 怎么做的

商机透镜不预测未来，它做的是「商业逻辑链 + 反向拆假设 + 最小验证」。

**先分清用户和购买者**  
喜欢用的人，不一定是付钱的人。没有购买者，很多“有用工具”都只能停在好想法。

**再看预算和替代方案**  
用户今天有没有为这个问题花钱、花时间、承担风险，或者忍受损失？如果没有，付费意愿要打折。

**然后看获客路径**  
第一批 10 个用户从哪里来？如果连第一批人都找不到，商业逻辑还没有闭环。

**最后定验证动作**  
不是问“你觉得好吗”，而是卖一个模板、做一次手动服务、放一个带价格的页面，或者让目标用户拿真实资料试一次。

**再过五道否决门槛**
只有痛点、购买者、首批用户路径、价值捕获和短周期可测性全部成立，才允许进入 `proceed`。这里的 `proceed` 只代表值得验证，不代表商业已经成立。

保存结构化决策后，可以运行：

```bash
python scripts/validate_business_decision.py decision.json
```

## 适合谁

- **独立开发者**：做之前先判断这个工具有没有付费路径。
- **产品创作者**：区分有趣 idea 和真实商机。
- **咨询和交付团队**：把客户问题转成可销售的产品化服务。
- **内容和课程团队**：判断模板、训练营、资料库、诊断报告能不能卖。
- **创业早期团队**：在写代码前找出致命假设。

如果你还没有 idea，先用 `idea-spark`。如果已经判断值得试，下一步用 `maker-forge` 做 3 小时 MVP。

## 三件套关系

```text
idea-spark      灵感火花：从资料和问题里找小产品机会
business-lens   商机透镜：形成有证据边界的商业决策和验证实验
maker-forge     造物工坊：把通过判断的机会锻造成 3 小时 MVP
```

## 局限

诚实地说：

- **不是财务、法律或投资建议**：它做的是商业模型推理，不是专业意见。
- **有用不等于能赚钱**：商机必须有购买者、预算、获客和交付路径。
- **不默认联网做市场研究**：除非你明确要求，否则只基于你给出的信息判断。
- **不替代真实销售测试**：最好的验证不是点赞，而是有人付出钱、时间或真实数据。
- **默认不写文件**：除非你明确要求保存、落地或生成报告。

一个好的商业判断工具，不该只让你兴奋。它应该帮你更早发现哪里会断。
